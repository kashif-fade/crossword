# The Daily Duo

A two-player-friendly puzzle app: one crossword tab, one sudoku tab. Built with
Vite + React, deployed on Vercel, no backend — all state lives in the
browser.

## Features

**Crossword**
- Mini (5×5, hand-built) and Midi (11×11, a 50-puzzle library)
- Type to fill, arrow keys to move, tap a cell twice to switch direction
- Check / Reveal / Clear
- Prev / Next / Shuffle through the 50-puzzle library
- Tracks which puzzles you've solved before (✓ shown in the library nav)

**Sudoku**
- Generates a fresh, uniquely-solvable puzzle on demand — unlimited, not a fixed bank
- Easy / Medium / Hard (controls how many starting digits are given)
- Pencil marks (candidate notes): toggle "Notes" mode, or hold Shift while typing a number
- Entering a final digit auto-clears that number from candidate notes in the same row/column/box
- Tracks lifetime solved count

**Both**
- Progress is saved to `localStorage` — closing the browser and reopening resumes
  the same tab, same puzzle, same fill state
- Mobile-friendly: a hidden input triggers the iOS/Android on-screen keyboard
  when a crossword cell is tapped (plain `<div>` cells have nothing for
  mobile Safari to attach a keyboard to otherwise)
- Cherry-blossom-themed visuals (falling petal animation respects
  `prefers-reduced-motion`)

## Tech stack

- **Vite + React** (no framework beyond that — plain inline styles, no CSS
  framework)
- **Deployment**: Vercel, auto-deploys from the connected GitHub repo
- No backend, no database — `public/crosswords.json` is a static asset,
  and per-device progress lives in `localStorage`

## Project structure

```
puzzle-duo/
├── index.html              Font import, box-sizing reset, petal keyframes
├── src/
│   ├── main.jsx             Entry point
│   └── PuzzleDuo.jsx        Everything: theme, both puzzle components, storage
├── public/
│   └── crosswords.json      The 50-puzzle Midi library (grid + clues)
└── package.json
```

Everything UI-related lives in one file (`PuzzleDuo.jsx`) for simplicity,
given the project's size — theme tokens at the top, then Sudoku's generator,
then both puzzle components, then the app shell.

## Running locally

```bash
npm install
npm run dev
```

## Deploying

Push to the connected GitHub repo — Vercel auto-builds (`vite build`) and
deploys. No environment variables needed.

## How the crossword library was built

The 50 Midi puzzles in `public/crosswords.json` weren't hand-authored — they
were generated by a pipeline, run once, and the output committed as a static
file:

1. **Word list**: [Crossword-Nexus's collaborative word list](https://github.com/Crossword-Nexus/collaborative-word-list)
   (MIT-licensed, ~425,000 scored entries).
2. **Content filtering**: every entry was checked against the public
   [LDNOOBW](https://github.com/LDNOOBW/List-of-Dirty-Naughty-Obscene-and-Otherwise-Bad-Words)
   blocklist (exact match) plus a short list of additional unsafe substrings
   (slurs and explicit-content roots verified to have no legitimate-word false
   positives). Entries 7+ letters long were further restricted to words that
   appear in a real dictionary (`/usr/share/dict/words`), which filters out
   awkward multi-word concatenations the raw list included (e.g. `ADMITONE`,
   `BUMPSINTO`).
3. **Grid generation**: a backtracking solver (most-constrained-slot-first,
   with forward-checking) fills a symmetric 11×11 block pattern from the
   filtered word list. Ten distinct block patterns and randomized candidate
   ordering give the 50 puzzles real variety instead of near-duplicates.
4. **Clues**: mostly auto-generated from **WordNet** definitions (Princeton's
   lexical database, freely licensed for any use) via the `wn` command-line
   tool, picking the most frequent sense per word. About 80 words WordNet
   doesn't cover (proper nouns, abbreviations, prepositions) were clued by
   hand. A handful of WordNet's picks landed on an obscure or flat-out wrong
   sense of a word and were manually corrected after review.
5. **Verification**: every single clue's row/col/length was checked
   programmatically against the actual solved grid before shipping, to
   guarantee the clue and the answer actually match.

The generation scripts themselves aren't part of this repo (they were run
once, ad hoc, to produce the JSON) — regenerating or expanding the library
would mean re-running that pipeline rather than hand-editing the JSON.

## Known limitations

- **No cross-device sync.** Progress is `localStorage`-only, scoped to one
  browser on one device. Playing on a phone and a laptop means two separate,
  unlinked progress states. (A shared backend — e.g. Vercel + Redis, keyed
  per puzzle — was scoped out during planning but never built.)
- **Clue quality is uneven, not hand-polished.** The library was
  auto-generated and spot-checked, not exhaustively reviewed word-by-word.
  A few fill words in the Midi puzzles are weak (obscure abbreviations,
  crosswordese) rather than what a human constructor would choose.
- **Mini is a single hand-built puzzle**, not a library like Midi.
- Sudoku's "New puzzle" always generates fresh — there's no way to replay a
  specific past sudoku (unlike the crossword's Prev/Next/Shuffle library).
